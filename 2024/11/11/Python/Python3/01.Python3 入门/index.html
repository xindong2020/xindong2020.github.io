一、Python 解释器

  • Python 是一种高级计算机语言，设计它们的目的是为了方便程序员理解和使用。但严格意义上来说，计算机只能理解一种非常低级的计算机语言，也就是 机器语言。

  • 我们需要设计一种方法，将高级语言翻译成计算机可以执行的机器语言。目前，有两种方法可以实现，分别是 编译器 和 解释器。

  • 使用 编译器 将自身转换成机器语言的高级语言，通常称为 编译型语言；而使用 解释器 将自身转换成机器语言的高级语言，称为 解释型语言，Python 就是一种解释型编程语言。

  • 解释型语言 由 解释器 逐行对源码进行解释，一边解释一边执行，不会生成可执行程序。

  • Python 官方针对 不同平台（例：Windows、MacOS、Linux）开发了不同的解释器，这些解释器必须要遵守同样的语法，识别同样的函数，完成同样的功能。只有这样，同一份代码才能在不同平台上拥有相同的执行结果。

  • Python 解析器种类

    • CPython，C 语言开发的解释器，应用广泛的解释器。

    • IPython，基于 CPython 的一种 交互式 解释器。

    • PyPy，基于 Python 语言开发的解释器，解析效率低。

    • JPython，运行在 Java 平台的解释器，直接把 Python 代码编译成 Java 字节码执行。

    • IronPython，运行在微软 .Net 平台的解释器，直接把 Python 代码编译成 .Net 字节码执行。

  • Python 解析器下载

    • 官网：https://www.python.org/downloads/ $\rightarrow$ Download Python 3.11.4

  • Python 解析器安装

    • 双击 python-3.11.4-amd64.exe 安装包。

    • 勾选 Add python.exe to PATH，然后点击 Customize installation。

    • Python 最重要的两部分：Python 解释器 和 pip 包依赖管理工具。

    • 勾选 Install Python 3.11 for all users $\rightarrow$ 自定义安装路径 $\rightarrow$ Install $\rightarrow$ Close。

    • 测试安装成功：命令提示符 cmd 中输入 python，出现交互式语言即为安装成功，输入 exit() 退出。

二、Python 之禅（Zen of Python）

  • Python 之禅最早由 Tim Peters 在 Python 邮件列表中发表，它包含了影响 Python 编程语言设计的 19 条软件编写原则。

  • 执行 import this 会输出一段文字。作者 Tim Peters。

  • Python 之禅列出了 Python 设计的原则与哲学，有助于理解与使用这种语言。

    • 优美优于丑陋（Python 以编写优美的代码为目标）

    • 明了优于晦涩（优美的代码应当是明了的，命名规范，风格相似）

    • 简洁优于复杂（优美的代码应当是简洁的，不要有复杂的内部实现）

    • 复杂优于凌乱（如果复杂不可避免，那代码间也不能有难懂的关系，要保持接口简洁）

    • 扁平优于嵌套（优美的代码应当是扁平的，不能有太多的嵌套）

    • 稀疏优于稠密（优美的代码有适当的间隔，不要奢望一行代码解决问题）

    • 可读性很重要（优美的代码是可读的）

    • 即便假借特例的实用性之名，也不可违背这些规则（这些规则至高无上）

    • 不要包容所有错误，除非你确定需要这样做（精准地捕获异常，不写
      except:pass 风格的代码）

    • 面对不确定性，拒绝妄加猜测。

    • 任何问题应有一种，且最好只有一种，显而易见的解决方法。

    • 尽管这并不容易，除非你是荷兰人（这里的荷兰人 Dutch 是指 Python 之父 Guido van Rossum，生于荷兰哈勒姆）。

    • 做优于不做，然而不假思索就做还不如不做。

    • 如果你无法向人描述你的方案，那肯定不是一个好方案；反之亦然（方案测评标准）

    • 命名空间是个绝妙的理念，我们应该好好利用它。

3.2 缩进

  • 每条语句开头处的空白被用来计算该行的缩进等级，以决定语句段
    落的组织结构。

  • 缩进常用于标示不同的代码块，如：函数体、条件语句、循环语句和类等。

  • 每条语句的缩进量是任意的（常用 4 个空格，即单个 Tab 键），但是同一个代码块中的缩进量必须保持一致。

四、静\动态、强\弱类型语言

4.1 静、动态类型语言

  • 静态类型 的编程语言指的是在创建一份数据（变量、参数、返回值等）时需要 显式 指明该数据的类型。通常情况下，这些数据的类型一旦被定义，在程序的整个生命周期也不再改变。

    • 静态类型的编程语言有很多，常见的有 C/C++、Java、C#、Go 等。

  • 动态类型 的编程语言在创建数据时往往不需要指明该数据的类型，而且数据的类型也会 随着值的改变而改变，给数据赋予不同的值，数据将得到不同的类型。

    • 动态类型的编程语言会根据赋予数据的值进行猜测或者推断该数据的类型。

    • 常见的动态类型语言有 JavaScript、PHP、Python、Ruby 等。

  • 一般而言，在 编译期 就确定变量类型的是 静态类型语言，在 运行期 才确定变量类型的则是 动态类型语言。

  • 静态语言 为我们提供了一个更加严格的编程环境，并且通常会生成更加健壮的代码。动态语言 为编写代码带来了灵活性，同时也提高了编写代码的效率，但是如果你不注意数据的类型，可能就会导致代码更容易出错。

4.2 强、弱类型语言

  • 强、弱类型指的是编程语言如何处理运算过程中的值。当值的类型 不符合运算规则 时，编程语言究竟是做出一些猜测，临时转换值的类型以帮助人们解决问题，还是停止运行，提醒人们不应该这样做。

  • 对于 “强弱类型” 概念，它的核心问题是 “不同类型的变量是否允许隐式转化”。

  • 强类型语言 的典型特征：它们不会处理与数据类型定义明显矛盾的运算，而是把它标记为一个语法错误，并且停止程序运行，将错误抛出。

    • 通常人们认为 Java、C#、Python、Go 都是强类型语言。

  • 和强类型语言不一样，当我们执行一些与类型定义不匹配的运算时，弱类型语言 尝试提供帮助，它可能会对其中一些值的类型进行转换，让它符合当前运算。

五、Python 变量与常量

5.1 Python 变量的定义

  • 变量就是可以变化的量。

  • 变量需要 先定义再使用，解释器执行到变量定义的代码时会申请 内存空间 存放 变量值，然后将变量值的 内存地址 绑定给 变量名。

  • 变量名是存储数据的一块内存地址的 别名。

    • 变量 存储 的只是一个变量的值所在的 内存地址，而不是这个变量的 值本身。

  • 每个变量都拥有 独一无二 的名字，通过变量的名字就能找到变量中的数据。

  • 变量存储的数据是 临时 的。

5.2 Python 变量的赋值

  • 将数据放入变量的过程就叫做 赋值。

    • Python 使用等号 = 作为赋值运算符，用来将变量值的内存地址绑定给变量名，格式：name = value。

      • 其中 name 表示变量名，value 表示变量值，也就是内存地址对应存储的数据。

  • 变量的值可以随时被修改，只要重新赋值即可，可以将不同类型的数据赋值给同一个变量。

  • 变量名要遵守 Python 标识符命名规范，还要避免和 Python 内置函数及 Python 保留字重名。

  • 标识符 主要作用是作为变量、函数、类、模块及其他对象的名称，Python 标识符命名规范：

    • 标识符是由字母（A ~ Z 和 a ~ z）、数字和下划线（_）组成。

    • 标识符第一个字符不能是数字。

    • 标识符不能与 Python 中的 保留字（报错）和 内置函数名（不报错）相同。

    • 标识符严格区分大小写。

    • 以下划线开头的标识符通常有特殊含义，例：

      • 以 单下划线开头 的标识符（例：_width）表示不能直接访问的 类属性，其无法通过 from ... import * 的方式导入。

      • 以 双下划线开头 的标识符（例：__add）表示 类的私有成员。

      • 以 双下划线开头和结尾 的标识符（例：__init__），是 专用标识符。

  • Python 变量的 命名习惯：

    • Python 变量命名一定要做到 见名知义。

    • 当标识符用作 模块名 或 包名 时，应全部使用小写字母，尽量不使用下划线。

    • 当标识符用作 类名 时，应采用单词首字母大写的形式（大驼峰）。

    • 当标识符用作 函数名 或 变量名 时，应全部使用小写字母，多个单词之间使用下划线分割。

      • 也可采用第一个单词首字母小写，其余单词首字母大写的形式（小驼峰）。

    • 当标识符用作 常量 时，应全部使用大写字母，多个单词之间使用下划线分割。

5.3 引用计数

  • 在一些代码中，如果存在一些变量但是没有用，会造成内存空间，因此叫做垃圾，所以要回收。

  • 引用计数是一种最直观，最简单的垃圾收集技术。原理非常简单，每一个对象都包含了两个头部信息，一个是 类型标志符，标识这个对象的类型；另一个是 计数器，记录当前指向该对象的引用数目，表示这个对象被多少个变量名所引用。

  • CPython 使用引用计数来管理内存，所有 Python 脚本中创建的实例，都会有一个引用计数，来记录有多少个指针指向它。当引用计数只有 0 时，则会自动释放内存。

  • 引用计数是用来记录对象被引用的次数，每当对象被创建或者被引用时将该对象的引用次数加一，当对象的引用被销毁时该对象的引用次数减一，当对象的引用次数减到零时说明程序中已经没有任何对象持有该对象的引用，换言之就是在以后的程序运行中不会再次使用到该对象了，那么其所占用的空间也就可以被释放了了。

    • 引用计数 增加 的场景：

      • 对象被创建并赋值给某个变量，例：a = 'ABC'。

      • 变量间的相互引用（相当于变量指向了同一个对象），例：b=a。

      • 变量作为参数传到函数中。例：ref_method(a)。

      • 将对象放到某个容器对象中（列表、元组、字典）。比如：c = [1, a, 'abc']。

    • 引用计数 减少 的场景：

      • 当一个变量离开了作用域，例：函数执行完成时，执行函数前后的引用计数保持不变，这就是因为函数执行完后，对象的引用计数也会减少，如果在函数内打印，则能看到引用计数增加的效果。

      • 对象的引用变量被销毁时，例：del a 或 del b。注：如果 del a，再去获取 a 的引用计数会直接报错。

      • 对象被从容器对象中移除，例：c.remove(a)。

      • 直接将整个容器销毁，例：del c

      • 对象的引用被赋值给其他对象，相当于变量不指向之前的对象，而是指向了一个新的对象。

5.4 Python 常量

  • 变量是变化的量，常量则是不变的量。

  • Python 中没有使用语法强制定义常量，也就是说，Python 中定义常量 本质上 就是 变量。

  • 如果非要定义常量，变量名 必须 全大写。并且 Python 只制定了一个规范，即常量的值 不应该 被更改，但 不是不可以 更改，而没指定常量的语法，因此常量也是可以修改的，但不建议。
