一、面向对象相关术语

  • 类：用来描述具有相同的属性和方法的对象的集合。它定义了该集合中每个对象所 共有的 属性和方法。对象是类的实例。

  • 实例对象：通过类创建出的实例。对象包括两个数据成员（类属性和实例属性）和方法。

  • 实例化：创建 一个类的 实例对象 的过程。

  • 属性：类中的所有变量都被称为属性。

  • 方法：类中的所有函数通常称为方法。不过，和函数所有不同的是，类方法至少要包含一个 self 参数。

  • 方法重写：如果从父类继承的方法不能满足子类的需求，可以对其进行改写，这个过程叫方法的覆盖（override），也称为方法的重写。

  • 方法重载：类中支持多个同名函数，但是 参数的个数或类型要不同，在调用的时候，解释器会根据参数的个数或者类型，调用相应的函数.

  • 封装：将类的某些部分（属性、方法）隐藏起来，具有一定的保护作用，隐藏对象的属性和方法实现细节，仅对外提供公共的访问方式。

  • 继承：即一个派生类（derived class）继承基类（base class）的字段和方法。继承也允许把一个派生类的对象作为一个基类对象对待。

  • 多态：多态指的是一类事物有多种形态。一个抽象类有多个子类，因而多态的概念依赖于继承。

二、面向过程与面向对象

\quad程序员从面向过程的 执行者 转化成了面向对象的 指挥者。

2.1 面向过程编程（Procedure Oriented Programming）

  • 面向过程编程是一种 以过程为中心 的编程思想，它首先分析出解决问题所需要的步骤，然后用函数把这些步骤一步一步实现，在使用时依次调用，是一种基础的顺序的思维方式。

  • 面向过程编程主要关注 “怎么做”，强调的是功能行为，即完成任务的具体细节，以 函数 为最小单位。

  • 面向过程编程的重心在 算法。尼古拉斯·沃斯：“算法 + 数据结构 = 程序”。

  • 面向过程编程好处是开发的速度快，但由于整个程序耦合得非常紧，任何一个处理函数和数据都可能关联到其他地方，增加了后期修改和维护的难度（耦合度高，扩展力低）。

2.2 面向对象编程（Object Oriented Programming）

  • 面向对象编程是按照人们认识客观世界的思维方式，采用基于 对象（实体）的概念，以对象为中心，以类和继承为构造机制，来认识、理解、刻画客观世界并设计、构建相应的软件系统。

  • 面向对象编程主要关注 “谁来做”，强调的是具备了功能的对象，即完成任务的对象，以 类 / 对象 为最小单位。

  • 面向对象编程更侧重于 数据结构。

  • 面向对象编程方法直接把所有事物都当作独立的对象，处理问题过程中所思考的不再主要是怎样用算法来描述问题，而是直接考虑重现问题中 各个对象之间的关系。

  • 面向对象编程中的类和继承是适应人们一般思维方式的描述范式，方法是允许作用于该类对象上的各种操作。这种对象、类、消息和方法的程序设计范式的基本点在于对象的 封装性 和类的 继承性。通过 封装 能将对象的定义和对象的实现分开，通过 继承 能体现类与类之间的关系，以及由此带来的 动态绑定 和实体的 多态性。

  • 面向对象编程实现了代码的 高内聚（相关功能聚合） 和 低耦合（模块之间的关联度低）。

  • 面向对象编程具有以下优点：

    • 可重用性：可重用性是面向对象编程的一个核心思想。通过类的继承性，共享公有属性，简化了对象、类的创建代码量，增加了代码的重用性，提高开发效率。另外，重用经过测试的代码还可以使产生额外错误的可能性达到最小。

    • 可扩展性：可扩展性是对现代应用软件提出的又一个重要要求。类的继承性使类能反映现实世界的层次结构，多态性反映了现实世界的复杂多样。类的继承性和多态性使得面向对象编程具有良好的可重用性和可扩展性。新的功能可以很容易地加入到系统中，便于程序的修改。

    • 可管理性：通过封装性能够将代码与其处理的数据结合，方便管理。

四、类的对象

4.1 实例对象的创建

4.1.1 概述

  • 实例对象的创建又称为 类的实例化，语法：实例对象名 = 类名(参数)。

    • 如果构造函数仅有一个 self 参数，我们无需为 self 传参。

    • 如果构造函数有多个参数时，我们需要为构造函数传递参数，否则抛出异常 TypeError: __init__() missing * required positional arguments。

  • 每实例化一次类，就会返回一个新的实例对象。实例对象的 数据类型 就是所属的类。

4.1.2 实例化的过程

  • __new__ 方法负责对象的 创建，而 __init__ 方法负责对象的 初始化。

  • __new__ 方法是在我们 调用类名进行实例化时 自动调用的，__init__ 方法是在这个类的每一次 实例化对象之后 自动调用的。

（1）构造方法 __new__(cls, *args, **kwargs)

  • 每一次实例化类时，Python 首先调用 的是该类的 构造方法 __new__(cls, *args, **kwargs)，用于 新建 该类的 实例 self 并 返回。如果该类没有定义 __new__ 方法，则去 父类 中依次查找，直到 object 类（object 类是所有类的基类），然后。

    • 参数 cls 就代表着要实例化的类，此参数在实例化时由 Python 解释器自动提供，后面的参数直接传递给 __init__ 方法。
    • 实例化传入的其他实参传递给形参 *args 和 **kwargs，然后 __new__ 方法会根据形参 cls 接收到的那个类对象创建出一个对应的实例对象，并返回该对象。

  • __new__ 方法可以是 object 的，也可以是自己 重写 的，重点是最后必须要 return 这个创建好的对象的引用。

  • __new__ 方法始终都是类的 静态方法，即使没有被加上静态方法装饰器。

  • 如果要得到当前类的实例，必须 在当前类中的 __new__ 方法语句中调用当前类的父类的 __new__ 方法。事实上，如果当前类中没有重写 __new__ 方法，Python 默认就是调用该类的直接父类的 __new__ 方法来构造该类的实例。

  • __new__ 方法主要是当你继承一些不可变的 class（例：int、str、tuple）时， 提供给你一个自定义这些类的实例化过程的途径。

（2）初始化方法 __init__(self, *args, **kwargs)

  • 新建好实例对象后，类的 初始化方法 __init__(self, *args, **kwargs) 会接收这个实例 self 作为自己的第一个参数，然后依次传入 __new__ 方法中接收到的位置参数和关键字参数，对实例对象 self 进行初始化操作，即 定制实例属性。

    • self 只是一个参数名称，可以自定义修改。

    • 如果当前类中没有定义 __init__ 方法，Python 同样会去 父类 中依次查找，直到 object 类。

  • __init__ 方法 不能有返回值，因为 __new__ 方法中已经返回了一个实例对象 self。如果显式 return 了返回值，则抛出异常 TypeError: __init__() should return None, not '***'。

  • 如果 __new__ 方法 没有返回 当前类的实例，那么当前类的 __init__ 方法 不会被调用。

  • 如果 __new__ 方法返回其他类的实例，那么会调用被返回的实例对象对应类的构造方法。

  • __new__ 方法可以决定是否要使用当前类的 __init__ 方法，因为 __new__ 可以调用其他类的构造方法或者直接返回别的对象来作为当前类的实例。

4.2 实例对象的使用

  • 使用已创建好的实例对象，通过 实例对象.变量名 的格式访问或修改类中的实例属性。

  • 使用已创建好的实例对象，通过 实例对象.方法名(参数) 的格式调用类中的方法。

    • 另一种方法调用的方式：将 实例对象.方法名 赋值给一个 变量，然后在变量后添加 (参数) 来调用，即 变量(参数)。

五、类的成员

  • 定义类的时候，会创建属于类的作用域（局部作用域），类对象属于全局变量。

    • 类中的类属性、类方法、实例方法和变量等全部都会被定义好。

      • 方法同名时，后定义的会覆盖先定义的。

    • 方法中的属性、变量、嵌套的方法等此时没有被定义。

5.1 方法

  • 方法就是定义在类中的函数。
  • 注：类中存在的同名方法，后定义的方法会覆盖先定义的方法。
  • 类中的属性和方法是类定义时被定义好，实例属性在实例化对象时被定义好。

5.1.4 魔术方法

  • 在 Python 中，所有以双下划线 __ 包起来的方法，统称为 Magic Method（魔术方法），它是一种 官方定义好 的特殊方法。

  • 普通方法需要我们主动调用，而 一般情况下我们不主动调用魔术方法，在 特定情况下魔术方法会自动被调用。

  • 魔术方法在类或对象的 某些事件触发 后会自动执行，让类具有神奇的 “魔力”。如果希望定制特殊功能的类，那么就需要对这些方法进行重写。

    • 注：事件触发条件也是官方定义好的。

  • Python 中常用的运算符、for 循环、以及类操作等都是运行在魔术方法之上的。

5.2 属性

5.2.1 类属性

  • 类属性（类变量）：以 属性名 = 属性值 格式 定义在类体中且在方法体之外 的变量被称为类属性，类属性在所有实例化对象中是 共享 的，即类属性就是实例对象所 共有 的属性。

  • 类属性既可以使用 类名调用：类名.类属性名；也可以使用类的 实例对象调用：实例对象.属性名。推荐使用类名调用。

    • 实例对象.类属性名 的本质：type(对象名).类属性名

  • 当类属性的 值存在 时，使用 类名.类属性名 = 类属性值 相当于 修改类属性。

  • 通过 类名 访问一个 不存在 的 类属性 时，就是为类和对象 动态增加类属性。

  • 可以使用 del 关键字 动态删除类属性，格式：del 类名.类属性名。注：不能使用实例对象名删除类属性。

  • 类属性为所有实例化对象共有，通过类名 修改类属性 的值，会影响 所有实例化对象。

  • 通过类的实例化对象对类属性进行赋值，其本质不是修改类属性的值，而是为该对象定义新的实例属性，这种操作被称作 动态定义实例属性。

5.2.2 实例属性

  • 实例属性（实例变量）：以 self.属性名 = 属性值 格式 定义在方法体中 的变量被称为实例属性，实例属性需要使用 self.属性名 进行访问，它只属于调用的实例对象，即 实例属性是实例对象特有的。

  • 实例属性只能通过对象名访问（实例对象.属性名），无法通过类名访问。

  • 当实例属性的 值存在 时，使用 实例对象.属性名 = 属性值 相当于 修改属性。

    • 使用 实例对象.__init__([实参列表]) 可以同时修改所有属性的值。

  • 通过 实例对象名 访问一个 不存在 的 实例属性 时，就是为实例对象 动态增加实例属性。每个实例之间新增的实例属性是互不干扰的。

  • 当实例属性和类属性 同名 时，对象会 优先 调用 实例属性，而不是类属性（此时的类属性需要 类名.属性名 才可以访问）。

  • 可以使用 del 关键字 动态删除实例属性，格式：del 实例对象.属性名。

  • 通过对象 修改实例属性 的值，不会影响 同类的其它实例化对象，更不会影响同名的类属性。

5.2.3 属性相关内置函数

（1）setattr

  • 格式：setattr(object, name, value)，Python 内置函数，可以为 类 或 实例对象 object 的属性 name 设置属性值，其中 name 要求为 字符串形式，无返回值。例：setattr(stu, "age", 18)。

    • setattr(object, name, value) 等价于 object.name = value。

    • 如果 属性不存在，就是为类 / 实例对象动态 新增 类属性 / 实例属性。

（2）getattr

  • 格式：getattr(object, name[, default])，Python 内置函数，可以根据属性名 name 返回 对应的属性值，其中 name 要求为 字符串形式。例：getattr(stu, "age")。

    • getattr(object, name) 等价于 object.name。

    • 如果属性不存在，返回默认值 default，如果没有指定 default，则抛出异常 AttributeError: '***' object has no attribute '***'。

（3）delattr

  • 格式：delattr(object, name)，Python 内置函数，用于删除 类 或 实例对象 object 的属性 name，其中 name 要求为 字符串形式，无返回值。例：delattr(stu, "age")。

    • delattr(object, name) 等价于 del object.name。

    • 当 name 属性不存在时，delattr 函数会抛出异常 AttributeError: ***。

（4）hasattr

  • 格式：hasattr(object, name)，Python 内置函数，用于判断 类 或 实例对象 是否拥有属性 name，其中 name 要求为 字符串形式。如果拥有属性则返回 True，否则返回 False。例：hasattr(stu, "age")。

    • hasattr 函数是通过调用 getattr 函数，查看是否抛出 AttributeError 异常来实现的。
