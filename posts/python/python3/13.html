一、封装

  • 封装，即 隐藏对象的属性和方法，这样在使用此类时，将无法直接以 “实例对象.属性名”（或 “实例对象.方法名(参数)”）的形式调用这些属性（或方法），而 只能使用未隐藏的类方法间接操作这些隐藏的属性和方法。

  • Python 类中的变量或方法，只存在 公有（public）和 私有（private）两种访问权限，这 2 种权限的区别如下：

    • public：公有权限的变量或方法，在 类的外部、类的内部 和 子类 中，都可以正常访问。

    • private：私有权限的变量或方法，只能在 本类内部 使用，类的外部以及子类都无法使用。

  • Python 并没有提供 public、private 这些权限修饰符。为了实现类的封装，Python 采取了下面的方法：

    • 默认情况下，Python 类中的变量或方法都是公有（public）的，它们的名称前都没有下划线（__）。

    • 如果类中的变量或方法，其名称以 双下划线 “__” 开头，则该变量（方法）为 私有变量（私有方法），“__” 等同于 private。

  • 封装原理：Python 封装仅仅是一种 变形操作，即将类中所有双下划线开头的属性名或方法名（例：Test 类中 __age、__info(self)），自动变形 成：_类名__属性名 或 _类名__方法名 的形式（例：_Test__age、_Test__info(self)）。

    • 这种机制并没有真正意义上限制我们从外部直接访问属性，知道了类名和属性名 / 方法名就可以拼出名字：_类名__属性名 / _类名__方法名，然后进行访问了。

  • 不建议 使用类名和属性名 / 方法名拼出名字去直接使用使用属性 / 方法，建议 提供一个公有方法在类的外部 间接访问 私有属性 / 方法。

  • 以 单下划线 “_” 开头的变量或方法（例：_age、_info(self)）通常也被视为私有变量或私有方法，但这 仅仅只是一个约定，Python 解析器并不会真的阻止用户去访问这个变量或方法，即以 “_” 开头的变量或方法也能通过实例对象正常访问。

  • 使用 from 模块名 import 私有变量/方法/类 的方式可以导入私有变量 / 方法 /类（_或__开头均可）并使用。但是 from 模块名 import * 的方式不会将私有类型导入。

二、继承

2.1 继承机制

2.1.1 概述

  • 继承经常用于创建和现有类功能相似的 新类，又或是新类只需要在 现有类 基础上 添加 一些成员（属性 / 方法），但又不想直接将现有类代码复制给新类。也就是说，通过使用继承这种机制，可以轻松实现类的重复使用，节省代码。

  • 继承的最大 意义 就是子类可以使用父类的所有公有属性与公有方法。从 子类 角度，继承 节省代码；从 父类 角度，继承可以 扩展功能。

  • Python 中，实现继承的类称为 子类（派生类），被继承的类称为 父类（基类、超类）。

  • 当子类的属性或方法不存在时，Python 解释器会自动去它的父类中查找。

    • 当调用父类的实例方法时，传入的 self 是子类的实例对象。

    • 当调用父类的类方法时，传入的 cls 是子类本身。

  • 派生 和继承是一个意思，只是观察角度不同：

    • 继承是相对于子类来说的，即子类继承自父类。

    • 派生是相对于父类来说的，即父类派生出子类。

2.1.3 多重继承

  • Python 的继承是 多重继承 机制，即一个子类可以同时拥有多个直接父类。

2.3 方法重写

2.3.1 概述

  • Python 子类可以继承父类中的方法，而不需要重新编写相同的方法。但有时子类并不想原封不动地继承父类的方法，而是想作一定的 修改，或者父类方法的功能不能满足需求，这时就需要采用方法的重写。方法重写又称方法覆盖。

  • 方法的重写是指两个方法的 返回值、方法名、参数的类型和个数 相同。

  • 方法的重写，不能发生在同类中，只能发生在 子类 中。

2.2.2 调用父类中被重写方法

（1）父类名.被重写方法

  • Python 中的类可以看做是一个 独立空间，而类方法就是处于该空间中的一个函数。如果想要全局命名空间中，调用类空间中的函数，只需要在调用该函数时 备注类名 即可。

  • 父类名.被重写方法 这种方式适用于 单继承。

  • 示例：Bird.fly(ostrich)，其中 Bird 为父类，手动为 self 参数赋值子类对象 ostrich，调用的是父类 Bird 中的 fly 方法。

（2）super 类

  • 可以使用 super() 函数解决多继承多次调用这个问题。在一个类中调用 super() 函数可以获取当前类在 MRO 列表中的下一个类，MRO 列表中不会出现多次相同的类。

  • super() 函数的 工作原理：

    • super() 函数通过 type2 获取 MRO 列表，然后在这个列表中查找 type1 类，返回类 type1 后面一个类的信息。

  • super 其实本身就是一个 类 class super(object)，super() 就是在对这个类进行 实例化，调用类的 初始化方法 __init__(self, type1=None, type2=None)。

    • __init__ 方法可以接收两个参数：要求 type1 是一个 类，type2 可以是一个 类 或 实例对象。

      • type2 提供 MRO 列表，type1 提供 MRO 搜索的起点。

    • super 类的实例化后返回一个 超类对象（super object），它的格式为 <super: <class '*'>, <* object>>，它是一个动态对象，它会根据调用顺序和方法解析顺序（MRO）动态计算。

      • <class '*'> 中的 * 表示 super 所在方法的类

  • 根据参数的 数量 和 类型，super() 可以有以下 4 种情况：

    • 无参形式 super()：

      • 无参形式的 super() 函数等价于 super(__class__, <first argument>)，本质上是一个二参形式。

        • __class__ 代表调用 super() 函数的方法所在类。

          • __class__ 出现在类的内部定义方法之外就是 not defined 的状态，所以也说明无参的 super() 只能出现在类的内部定义方法之内。

        • <first argument> 是指调用 super() 函数的方法的第一个参数。

          • 类方法中就是 cls，实例方法中就是 self。

          • 类方法第一个参数一定是类本身 cls，那么调用 super() 就相当于调用 super(__class__, cls)。因此在类方法中使用 super()，就相当于调用父类中的类方法。

      • 调用 super() 时用到 <first argument> 的 MRO 列表，在这个列表中查找 __class__ 类，返回该类后面一个类的实例对象。

    • 二参形式 super(type, obj)：

    • 这种形式要求两个参数 type 类对象和 obj 实例对象必须符合 isinstance(obj, type)==True，即 obj 必须是 type 的 实例对象。

      • 注：子类的实例对象也是父类的实例对象，即 isinstance 返回 True。

    • 调用 super(type, obj) 时用到 obj 的 MRO 列表，从指定 type 的下一个类开始查找，看看是否有指定的方法，如果有，就将 obj 当作调用方法的第一个自变量。

    • 二参形式 super(type1, type2)：

      • 这种形式要求两个参数 type1 类对象 和 type2 类对象 必须符合 issubclass(type2, type1)==True，即 type2 必须是 type1 的 子类。

        • 当 type1 == type2 时，issubclass 返回 True，表示调用自己父类中的类方法。

  • 注：super() 函数和父类并没有直接关联，super 获取到的也可能不是父类。例：

    • 我们会发现类 A 和类 B 没有继承关系，可是类 A 中的 super() 却调用了类 B 中的 func 方法。这是因为类 C 的 MRO 列表为：

      • 类 C 中的 super() 等价于 super(C, C())，获取到类 C 在 MRO 列表中的下一个类 A，因此 super().func() 等价于 A().func()。

      • 类 A 中的 super() 等价于 super(A, A())，获取到类 A 在 MRO 列表中的下一个类 B，因此 super().func() 等价于 B().func()。

        • 这里也就解释清楚了类 A 和类 B 没有继承关系，可是类 A 中的 super() 却调用了类 B 中的 func 方法。

2.4 继承的 __init__

  • 如果子类 未重写 __init__ 方法，那么子类 继承 父类的 __init__ 方法，实例化子类时，会 自动调用 父类中定义的 __init__ 方法。

    • 注：当子类继承多个父类时，只有第一个父类中的 __init__ 方法会被调用。

  • 如果子类 重写 了 __init__ 方法，实例化子类时，就 不会调用 父类中已经定义的 __init__ 方法。

    • 如果想要在子类中调用父类中的 __init__ 方法，需要 显式调用，有两种方式：

      • 父类名.__init__(self, 参数列表)

      • 推荐：super(子类, self).__init__(参数列表) 或 super().__init__(参数列表)

2.5 继承相关的内置函数

2.5.1 isinstance

  • 格式：isinstance(object, classinfo)，用于判断一个 object 实例对象是否是给定的 classinfo 或其 子类 的实例对象，如果是则返回 True，反之返回 False。classinfo 可以是类，也可以是由类组成的元组。

2.5.2 issubclass

  • 格式：issubclass(class, classinfo)，用于判断一个类 class 是否是给定的 classinfo 的子类。classinfo 可以是类，也可以是由类组成的元组。

  • 判断规则：

    • 一个类是自己的子类。

    • 一个类直接或间接继承自另一个类，都被认为是子类。

    • classinfo 可以是一个元组，class 是其中一个类的子类（满足前两条），函数就返回 True。

三、多态

3.1 概述

  • 多态是指一类事物有多种形态，例：动物类，可以有猫，狗等。

  • 一个抽象类有多个子类，因而多态的概念依赖于继承。

  • 类的多态特性，需要满足以下 2 个 前提条件：

    • 继承：多态一定是发生在子类和父类之间。

    • 重写：子类重写了父类的方法。

  • 当子类和父类都存在 相同 的方法时，子类的方法 覆盖 了父类的方法。
