一、os 模块：实现 Python 程序与操作系统进行交互

  • os 就是 “operating system” 的缩写，顾名思义，os 模块提供的就是各种 Python 程序与操作系统进行交互的接口。通过使用 os 模块，一方面可以方便地与操作系统进行交互，另一方面也可以极大增强代码的可移植性。如果该模块中相关功能出错，会抛出 OSError 异常及其子类异常。

  • 不要使用 from os import * 来导入 os 模块；否则 os.open() 将会覆盖内置函数 open()。

  • os 模块中大多数接受路径作为参数的函数也可以接受 “文件描述符” 作为参数。

    • 文件描述符：file descriptor，在 Python 文档中简记为 fd，是一个与某个打开的文件对象绑定的非负整数，打开已有文件或新建文件时，系统内核会返回一个文件描述符。

1.1 属性

  • os.name 属性返回当前操作系统的类型，返回值 posix，nt，java 分别对应 linux/Mac OS，Windows，Java虚拟机。

  • os.environ 属性返回 字典类型 的环境变量。

1.2 路径相关操作

1.2.1 os.getcwd()

1.2.2 os.listdir(path)

  • listdir 是 list directories 的缩写，用于返回一个由指定文件夹目录（path）下的文件和文件夹名字组成的 列表。注：只返回指定文件夹下的名称，不包含子目录下的名称。

    • path 表示需要列出的目录路径，可以是相对或绝对路径。

  • 示例：

1.2.3 os.mkdir(path, mode=0777)

（1）用法

  • mkdir 是 make directory 的缩写，用于以数字权限模式创建目录（单级目录），无返回值。

    • path 表示要创建的目录，可以是相对或者绝对路径。

    • mode 表示要为目录设置的数字权限模式。默认为 0777 (0 开头表示八进制)。

  • mkdir 只会创建 path 中的 最后一级目录，否则（即新建多级路径）会抛出 FileNotFoundError 异常。所以要保证所需要的中间目录是存在的，如果最后一级目录已存在，则抛出 FileExistsError 异常。

  • 示例：

（2）权限位说明

  • 文件或目录的权限位是由 9 个权限位来控制，每三位为一组，它们分别是文件所有者（user）的读、写、执行，用户组（group）的读、写、执行和其他用户（other）的读、写、执行，所有用户（all）的读、写、执行。

    • r read 可读权限 对应数字 4。

    • w write 可写权限 对应数字 2。

    • x excute 执行权限 对应数字 1。

    • - 没有任何权限 对应数字 0。

1.2.4 os.makedirs(path, mode=0777, exist_ok=False)

  • os.makedirs() 方法用于递归创建目录。在需要新建多级目录的场景下，就可以使用 os.makedirs()，无返回值。如果 exist_ok 为 False（默认值），则子目录创建失败或已经存在时，会抛出一个 FileExistsError 的异常，为 True 则不会抛出异常。

  • os.remove() 方法用于删除指定路径下的文件，无返回值。如果指定的路径是一个目录，将抛出 PermissionError 异常。

1.2.6 os.rmdir(path)

  • os.rmdir() 方法用于删除指定路径的目录，无返回值。仅当这个文件夹 为空 时才可以被删除，否则抛出 OSError 异常。目录不存在则抛出 FileNotFoundError 异常。

  • os.path.dirname() 方法用于返回路径 path 的目录名称。即除了最后一级名称之外的全部名称。

  • os.path.split() 方法用于将路径分割成 dirname 和 basename，返回一个 元组。

  • os.path.exists() 方法用于判断路径是否存在，path 路径存在则返回 True，不存在或失效则返回 False。

  • os.path.exists() 方法用于判断路径 形式上 是否为绝对路径，不会判断是否存在。返回 True 或 False。

  • os.path.isfile() 方法用于判断路径是否为文件，路径是文件且 文件存在 时返回 True，否则返回 False。

1.2.18 os.path.isdir(path)

  • os.path.isdir() 方法用于判断路径是否为目录，路径是目录且 目录存在 时返回 True，否则返回 False。

  • 示例：

1.2.19 os.path.join(path, *paths)

  • os.path.join() 方法可以将多个传入路径组合为一个路径。

    • 该方法是将传入的几个字符串用系统的 分隔符（\）拼接 起来，组合成一个新的字符串，所以一般的用法是将第一个参数作为父目录，之后每一个参数作为下一级目录，从而组合成一个新的符合逻辑的路径。

    • 如果传入路径中存在一个 “绝对路径” 格式的字符串，且这个字符串不是函数的第一个参数，那么在这个参数之前的所有参数都会被丢弃，余下的参数再进行拼接。更准确地说，只有最后一个 “绝对路径” 及其之后的参数才会体现在返回结果中。

2.1 pickle 模块：实现 Python 对象的持久化存储

  • pickle 模块能够实现任意对象与文本之间的相互转化，也可以实现任意对象与二进制之间的相互转化。也就是说，pickle 可以实现 Python 对象的存储及恢复。

  • pickle 模块提供了 dumps()、loads()、dump() 和 load() 这 4 个函数供我们使用。

    • dumps 和 loads 函数实现了基于内存的 Python 对象与二进制之间的相互转化。

    • dump 和 loads 函数实现了基于文件的 Python 对象与二进制之间的相互转化。

2.1.1 pickle.dumps() 函数

  • pickle.dumps() 函数用于将 Python 对象转换为二进制对象，返回值为二进制对象。语法格式：dumps(obj, protocol=None, *, fix_imports=True)。

    • obj：表示要转换的 Python 对象。

    • protocol：pickle 的转码协议，取值为 0、1、2、3、4，其中 0、1、2 对应 Python 早期的版本，3 和 4 则对应 Python 3.x 版本及之后的版本。未指定情况下，默认为 3。

    • 其它参数：为了兼容 Python 2.x 版本而保留的参数，Python 3.x 中可以忽略。

5.1.2 pickle.loads() 函数

  • pickle.loads() 函数用于将二进制对象转换成 Python 对象，返回值为 Python 对象。语法格式：loads(data, *, fix_imports=True, encoding='ASCII', errors='strict')。

    • data：表示要转换的二进制对象。

    • 其它参数：为了兼容 Python 2.x 版本而保留的参数，Python 3.x 中可以忽略。

  • 示例：

5.1.3 pickle.dump() 函数

  • pickle.dump() 函数用于将 Python 对象转换成二进制文件，无返回值。语法格式：dump(obj, file, protocol=None, *, fix mports=True)

    • obj：表示要转换的 Python 对象。

    • file：转换到指定的二进制文件中，要求该文件必须是以 “wb” 的打开方式进行操作。

    • protocol：和 dumps() 函数中 protocol 参数的含义完全相同，因此这里不再重复描述。

    • 其它参数：为了兼容 Python 2.x 版本而保留的参数，Python 3.x 中可以忽略。

  • 示例：

5.1.4 pickle.load() 函数
