Python3 模块、包和库

一、模块（.py 文件）

1.1 模块定义

  • 模块（Modules）是使用 Python 实现某一特定功能，包含所有定义的函数和变量的代码文件，其文件名是模块名加后缀名（.py）。

  • 模块的类型为 'module'，即使用 type(模块名) 的返回值为 <class 'module'>。

1.2 import 使用

  • 在 Python 编程时，有些功能可以借助 Python 现有的标准库或者其他人提供的第三方库来实现。使用这些第三方库之前需要使用 import 关键字将其导入到自己的模块中。

  • 当导入某个模块时，被导入的模块会被执行后再导入。

  • 一个模块只会被另一个程序导入一次，也只会被执行一次。

  • 使用 import 导入的语法主要有两种：

    • 方式一：import 模块名1 [as 别名1], 模块名2 [as 别名2], ...

      • 可以直接导入 整个模块，不设置别名。例：import math。

      • 导入 整个模块 时，也可以为模块指定别名。例：import math as m。

      • import 支持一次导入多个模块，多个模块之间用逗号隔开。例：import math, os。

      • import 导入多个模块时，也可以为模块指定别名。例：import math as m, os as o。

      • 使用这种语法格式的 import 语句，会导入指定模块中的所有成员（变量、函数、类等）。

      • 当需要使用模块中的成员时，需用该模块名（或别名）作为前缀，否则 Python 解释器会报错。例：math.pi（或m.pi）。

    • 方式二：from 模块名 import 成员名1 as [as 别名1], 成员名2 [as 别名2], ...

      • 可以直接导入 模块成员，不设置别名。例：from math import pi。

      • 导入模块成员时，也可以为成员指定别名。例：from math import pi as p。

      • form...import... 支持一次导入多个成员，多个成员之间用逗号隔开。例：from math import pi, inf。

      • form...import... 导入多个成员时，也可以为成员指定别名。例：from math import pi as p, inf as i。

      • 使用 form...import... 语句会导入指定模块中的指定成员（变量、函数、类等）。当需要使用模块中的成员时，不需要再添加前缀。

      • from...import 可以一次导入指定模块内的所有成员。例：from math import *。

        • 不推荐使用这种方式，因为当同时导入两个模块内的所有成员时，如果存在同名函数，程序运行时会报错。

  • Python 内置函数 dir() 可以查看模块内定义的所有成员（变量、函数 和 类），以字符串列表的形式返回。这里所指的所有成员，不仅包含外部可调用的模块成员，还包含所有名称以双下划线 “__” 开头和结尾的成员，而这些 “特殊” 命名的成员，是为了在本模块中使用的，并不希望被其它文件调用。

1.4 模块查找路径

  • 对于用 import 语句导入的模块，Python 会按照以下顺序查找指定的模块文件：

    • 执行 import 语句的文件所在路径。

    • 项目的根目录（pycharm 会默认将项目的根目录添加到 sys.path 中）。

    • Pycharm 相关路径。

    • python path 环境变量的每一个目录。

      • 查询 pycharm 的 python path 环境变量：File →\rightarrow→ Settings... →\rightarrow→ Project:XXX →\rightarrow→ Python Interpreter →\rightarrow→ 设置 →\rightarrow→ show all →\rightarrow→ Show paths for the selected interpreter

        • Lib 目录：内置模块、包存放路径。

        • Lib\site-packages 目录：下载的第三方库存放路径。

    • 当前 Python 解释器所在路径。

  • 对于用 import 语句导入的模块所查找的路径，可以通过标准模块 sys 的 sys.path 变量进行查询（返回一个 列表）。换句话说，如果要导入的模块没有存储在 sys.path 显示的目录中，那么导入该模块并运行程序时，Python 解释器会抛出 ModuleNotFoundError（未找到模块）异常。

  • 解决 ModuleNotFoundError 异常的方法有 3 种，分别是：

    • 使用 sys.path.append 函数向 sys.path 变量中 临时添加 模块文件所在的完整路径。

      • 注：路径中的 ‘\’ 需要使用 r 保持原始字符串或使用 \ 进行转义，否则会导致语法错误。

    • 将模块移动到 sys.path 变量中已包含的路径中。

    • 将模块所在路径加入到 path 环境变量。

1.5 __all__属性

  • 当我们向文件导入某个模块时，导入的是该模块中不以下划线（单下划线 “_” 或 双下划线“__”）开头的变量、函数和类。因此，如果我们不想自定义模块中的某个成员被引入到其它文件中，可以在其名称前添加下划线。

  • 除此之外，我们还可以借助模块提供的 __all__ 属性，该变量的值是一个列表，其中存储的是当前模块中一些成员（变量、函数或者类）的名称。通过在模块文件中设置 __all__ 属性，当其它文件以 “from 模块名 import *” 的形式导入该模块时，该文件中只能使用 __all__ 列表中指定的成员。

  • __all__ 属性仅限于在其它文件中以 “from 模块名 import *” 的方式引入。如果使用以下 2 种方式引入模块，则 __all__ 属性的设置是无效的。

    • 以 “import 模块名” 的形式导入模块。通过该方式导入模块后，总可以通过模块名（别名）前缀来调用模块内的所有成员（除了以下划线开头命名的成员）。

    • 以 “from 模块名 import 成员” 的形式直接导入指定成员（非下划线开头命名的成员）。使用此方式导入的模块，__all__ 属性的设置无效。

  • 除了上述功能之外，还可以使用 __all__ 属性查看模块内包含的所有成员。和 dir() 函数相比，__all__ 属性在查看指定模块时，它不会显示模块中的特殊成员，同时还会根据成员的名称进行排序显示。

1.6 __doc__属性

  • __doc__ 是 python 模块的内置属性，使用 模块名.__doc__ 可以返回模块说明文档（本质是字符串，位于该成员内部开头的位置）。如果模块是包的话，则返回包的 __init__.py 文件的说明文档。

  • 使用内置函数 help(object) 同样可以查看模块说明文档，help() 函数底层是借助 __doc__ 实现的。

1.7 __file__属性

  • __file__ 是 python 模块的内置属性，使用 模块名.__file__ 可以返回模块文件的绝对路径，如果模块是包的话，则返回包的 __init__.py 文件的绝对路径。

  • 并不是所有模块都提供 __file__ 属性，因为并不是所有模块都采用 Python 实现，有些模块采用的是其它编程语言（例：C 语言）。

二、包（存放多个模块的文件夹）

2.1 包的概述

  • 包是存在一个名为 “__init__.py” 的文件（Python 3 中非必须）的文件夹。

    • 当一个包被导入（import）时，__init__.py 文件会自动被执行。

  • 包的本质还是模块，因此使用 type(包名) 的返回值为 <class 'module'>。

  • 和文件夹一样，包里面还可以装其他的包。

2.2 包的作用

  • 避免相同命名冲突：在同一个包内，不允许两个模块命名相同，但是不在同一个包中是可以的。

  • 模块分区：把不同功能的模块归类到不同的包中，方便查询和修改。在比较大型的项目中需要编写大量的模块，此时我们可以使用包来对这些模块进行管理。

2.3 创建包

  • 创建一个包，需进行以下 2 步操作：

    • 新建一个文件夹，文件夹的名称就是新建包的包名。

    • 在该文件夹中，创建一个 __init__.py 文件（前后各有 2 个下划线 “_”），该文件中可以不编写任何代码。当然，也可以编写一些 Python 初始化代码，当有其它程序文件导入包时，会自动执行该文件中的代码。

  • 创建包后，可以向包中继续添加模块（也可以添加包）。

2.4 导入包

  • 包的本质还是模块，因此导入模块的语法同样也适用于导入包。

2.4.1 import 包名.模块名 as 别名

  • import 包名[.模块名 [as 别名]]，用 [] 括起来的部分是可选部分。

  • 通过此语法格式导入包中指定模块后，在使用该模块中的成员（变量、函数、类）时，需添加 “包名.模块名（别名）” 作为前缀。

  • 当导入指定模块时，程序会自动执行该包所对应文件夹下的 __init__.py 文件中的代码。

  • 直接导入包名，并不会将包中所有模块全部导入到程序中，它的作用仅仅是导入并执行包中的 __init__.py 文件，后续使用包中成员时会抛出 AttributeError 异常。

  • import 包名.* 会抛出 SyntaxError: invalid syntax 异常。

2.4.2 from 包名 import 模块名 as 别名

  • from 包名 import 模块名 [as 别名]，使用 [] 括起来的部分是可选部分。

  • 通过此语法格式导入包中指定模块后，在使用该模块中的成员时不需要添加包名前缀，但需要添加 “模块名（别名）” 作为前缀。

  • from 包名 import * 与 import 包名 的作用一样，都只是将该包中的 __init__.py 文件导入并执行，后续使用包中成员时会抛出 AttributeError 异常。

2.4.3 from 包名.模块名 import 成员名 as 别名

  • from 包名.模块名 import 成员名 [as 别名]，使用 [] 括起来的部分是可选部分。

  • 通过此语法格式导入 “包名.模块名” 中的指定成员（变量、函数或类），在使用该成员时可以直接使用成员的名字（变量名、函数名或类名）进行调用，不需要添加任何前缀。

  • 通过此种语法格式导入指定包中的指定模块时，可以使用 * 代替该模块中的成员名，表示加载该模块下的所有成员。

2.4.4 __all__属性

  • __all__ 属性可以用于模块中，__init__.py 本质也是一个模块，在包的 __init__.py 中使用 __all__ 属性可以指定使用 “from 包名.模块名 import *” 的形式导入包中模块时，只能导入 __all__ 列表中指定的模块。

三、库

  • 在 Python 中，具有某些 相关功能 的模块和包的 集合 都可以被称作库。

  • Python 标准库就是下载安装的 python 中 自带 的所有模块的总称，可以称为 第一方库。

  • 第二方库 就是我们 自己编写 的所有模块的总称。

  • 第三方库 就是 别人提供 的模块的总称。我们可以通过下列三种方式安装（卸载）第三方库：

    • 方式一：使用 PyCharm 进行安装（卸载）

      • 安装：File →\rightarrow→ Settings... →\rightarrow→ Project:XXX →\rightarrow→ Python Interpreter →\rightarrow→ 关闭 ues Conda Package manger （绿色按钮）→\rightarrow→ 加号 + →\rightarrow→ 搜索要下载的库名 →\rightarrow→ install package

        • 第三方库安装位置为：..\anaconda\envs\py39torch\Lib\site-packages

        • 点击下方 Manage Repositories 可以更换国内源（选择下方一个）：

          • 清华：https://pypi.tuna.tsinghua.edu.cn/simple

          • 豆瓣：http://pypi.douban.com/simple/

          • 阿里：http://mirrors.aliyun.com/pypi/simple/

          • 腾讯：http://mirrors.cloud.tencent.com/pypi/simple

      • 卸载：File →\rightarrow→ Settings... →\rightarrow→ Project:XXX →\rightarrow→ Python Interpreter →\rightarrow→ 关闭 ues Conda Package manger （绿色按钮）→\rightarrow→ 选中要删除的库名 →\rightarrow→ 减号 +

    • 方式二：使用命令行安装（卸载）

      • 安装：打开 anaconda 的命令行 “Anaconda Prompt (anaconda)” 或 “Anaconda Prompt (虚拟环境)”，输入 pip install PackageName 或 conda install PackageName，即可安装第三方库。

        • “Anaconda Prompt (anaconda)” 安装位置为：..\anaconda\Lib\site-packages

        • “Anaconda Prompt (虚拟环境)” 安装位置为：..\anaconda\envs\虚拟环境名\Lib\site-packages

      • 卸载：pip uninstall PackageName 或 conda uninstall PackageName。

    • 方式三：先把要安装的第三方库文件（.whl 后缀文件）下载到本地再进行安装。

      • 推荐网址：https://www.lfd.uci.edu/~gohlke/pythonlibs/， 打开网站 ctrl + F 搜索要下载的库。

      • 打开 anaconda 的命令行 “Anaconda Prompt (anaconda)” 或 “Anaconda Prompt (虚拟环境)”，输入 pip install .whl文件所在路径 + whl文件名 或 conda install .whl文件所在路径 + whl文件名，即可安装第三方库。例：pip install D:\GoogleDownload\wordcloud-1.8.1-cp39-cp39-win_amd64.whl

        • “Anaconda Prompt” 中安装的第三方库，都会被安装在 ..\Lib\site-packages 文件夹中。
