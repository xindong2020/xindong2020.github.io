一、正则表达式

1.1 概述

  • 正则表达式，又称规则表达式（Regular Expression），一个正则表达式通常被称为一个模式（pattern），是用来描述、匹配某个 句法规则 的 字符串，常被用来检索、替换那些符合某个模式（规则）的文本。
  • 执行匹配时，是字符串匹配正则表达式，而不是正则表达式匹配字符串。
  • 正则表达式匹配完全了，才算是一次成功的匹配。
  • 因为正则表达式和 Python 字符串都使用反斜杠字符（\）来转义，有时就需要使用更多的反斜杠才能达到想要的效果，而这很麻烦，所以建议在书写正则表达式时，使用原始字符串。
    • 注：在正则表达式中，四个反斜杠（\\\\）才会表示一个反斜杠（\）。
      • 解析：Python 字符串中反斜杠需要转义，因此使用两个反斜杠才会表示一个反斜杠，而在正则表达式的 compile 方法中，反斜杠同样需要进行转义，因此四个反斜杠才会表示一个反斜杠。

1.2 语法

  • 完整的正则表达式由两种字符构成：特殊字符（元字符）和 普通字符。

1.2.1 普通字符

  • 普通字符包括没有显式指定为元字符的所有可打印和不可打印字符。
  • 注：普通字符只会和自身匹配。

1.2.2 特殊字符（元字符）

  • 一共含有 11 个特殊字符：., ^, $, *, +, ?, {}, [], \, |, ()
（1）基本元字符 . | [ ] \ ( )

（2）数量元字符 { } * + ?

  • 贪婪匹配：正则表达式通常的行为是（在使整个表达式能得到匹配的前提下）匹配 尽可能多 的字符。例：a.*b，它将会匹配 最长 的以 a 开始，以 b 结束的字符串。如果用它来搜索 aabab 的话，它会匹配整个字符串 aabab。

    • {n}：匹配前面的子表达式 确定的 n 次，n 是一个非负整数。例：o{2} 不能匹配 Bob 中的 o，但是能匹配 food 中的两个 o。

      • 示例：

    • {n,m}：匹配前面的子表达式最少 n 次且最多 m 次，m 和 n 均为非负整数，其中 n <= m。例：o{1,3} 将匹配 fooooood 中的前三个 o。

      • 注：在逗号和两个数之间不能有空格。

      • 示例：

    • {n,}：匹配前面的子表达式 n 次或 多次，n 是一个非负整数。例：o{2,} 不能匹配 Bob 中的 o，但是能匹配 foooood 中的所有 o。

      • 示例：

    • {,m}：匹配前面的子表达式最少 0 次且最多 m 次，n 是一个非负整数。例：o{2,} 不能匹配 Bob 中的 o，但是能匹配 foooood 中的所有 o。

      • 示例：

    • *：匹配前面的子表达式 零次 或 多次，等价于 {0,}。例：zo* / zo{0,} 能匹配 z、zo、zoo 等。* ，即 zo 等价于 zo{0,}。

      • 示例：

    • +：匹配前面的子表达式 一次 或 多次，等价于 {1,}。例：zo+ / zo{1,} 能匹配 zo、zoo 等，但是不能匹配 z。

      • 示例：

    • ?：匹配前面的子表达式 零次 或 一次，等价于 {0,1}。例：do(es)? / do(es){0, 1} 可以匹配 “do” 、"does"、"dong"。

      • 示例：

  • 惰性匹配：惰性匹配也就是 匹配尽可能少 的字符，在能使整个表达式匹配成功的前提下使用最少的字符进行匹配。只要在贪婪匹配的元字符后面加上一个问号 ? 即可变成惰性匹配元字符。例：a.*b，它将会匹配 最长 的以 a 开始，以 b 结束的字符串。如果用它来搜索 aabab 的话，它会匹配整个字符串 aabab。例：a.*?b，它将会匹配 最短 的以 a 开始，以 b 结束的字符串。如果用它来搜索 aabab 的话，它会匹配字符串 aab。

    • *?、+?、?? 都是惰性匹配。

（3）非打印字符

  • 非打印字符指在计算机中有一些字符是确确实实存在,但是它们不能够显示或者打印出来。以 ASCII 码表为例，ASCII 码值在 0 ~ 31 的为控制字符，无法显示和打印。

    • \cx：匹配由 x 指明的 控制字符。x 的值必须为 A-Z 或 a-z 之一，否则将 c 视为一个原义的 c 字符。例：\cM 匹配一个 Control-M 或回车符。

    • \f：匹配一个 换页符。等价于 \x0c。

    • \n：匹配一个 换行符。等价于 \x0a。

    • \r：匹配一个 回车符。等价于 \x0d。

    • \s：匹配任何 空白字符，包括空格、制表符、换页符等。等价于 [\f\n\r\t\v]。

    • \S：匹配任何 非空白字符。等价于 [^\f\n\r\t\v]。

    • \t：匹配一个制表符。等价于 \x09。

    • \v：匹配一个垂直制表符。等价于 \x0b。

二、re 模块

2.1 概述

  • Python 自 1.5 版本起增加了 re 模块，使 Python 语言拥有全部的正则表达式功能。

  • Python 自带的 re 模块主要包含如下 6 种方法：

    • re.compile：编译一个正则表达式模式（pattern）。

    • re.match：从头开始匹配，使用 group() 方法可以获取第一个匹配值。

    • re.search：用包含方式匹配，使用 group() 方法可以获取第一个匹配值。

    • re.findall：用包含方式匹配，把所有匹配到的字符放到以列表中的元素返回多个匹配值。

    • re.sub：匹配字符并替换。

    • re.split：以匹配到的字符当做列表分隔符，返回列表。

2.2 re.compile 方法

  • 格式：re.compile(pattern[, flags])，用于编译正则表达式，生成一个正则表达式对象（Pattern），供 match() 和 search() 这两个函数使用。

    • pattern：一个字符串形式的正则表达式。

    • flags：可选参数，表示编译模式，用于控制正则表达式的匹配方式，参数值为：

      • re.I / re.IGNORECASE：忽略大小写。

      • re.M / re.MULTILINE：多行匹配，影响 ^ 和 $。

      • re.S / re.DOTALL：使 . 匹配包括换行符在内的所有字符，没有设置时 . 不能匹配换行符。

      • re.X / re.VERBOSE：为了增加可读性，允许正则表达式中书写空格、分段、注释。

      • re.L / re.IGNORECASE：表示特殊字符集 \w、\W、\b、\B、\s、\S 依赖于当前环境。

      • re.U / re.IGNORECASE：表示特殊字符集 \w, \W, \b, \B, \d, \D, \s, \S 依赖于 Unicode 字符属性数据库。

2.3 Pattern 实例对象方法

2.3.1 Pattern.match 方法

  • 格式：Pattern.match(string[, pos[, endpos]])，该方法尝试从字符串 string 的 pos 起始位置（默认起始位置）到 endpos 结束位置（默认为 len(string)）匹配一个模式。

    • 如果匹配成功，返回 Match 类的实例对象（该实例对象包含匹配相关的信息：起始和结束位置、匹配的子串等等）。
    • 如果字符串开始就不符合正则表达式 pattern，则匹配失败，函数返回 None。

  • 示例：

2.3.2 Pattern.search 方法

  • 格式：Pattern.search(string[, pos[, endpos])，该方法从字符串 string 的 pos 起始位置（默认起始位置）到 endpos 结束位置（默认为 len(string)）进行匹配。

    • 匹配成功则返回 第一个成功匹配 的 Match 类实例对象（该实例对象包含匹配相关的信息：起始和结束位置、匹配的子串等等）。
    • 匹配失败则返回 None。

  • re.search 与 re.match 区别：re.match 只匹配字符串的开始，如果字符串开始就不符合正则表达式，则匹配失败，函数返回 None；而 re.search 匹配直到找到第一个匹配的对象，最后找不到才返回 None。

  • 示例：

2.3.3 Pattern.fullmatch 方法

  • 格式：Pattern.fullmatch(string[, pos[, endpos]])，该方法从字符串 string 的 pos 起始位置（默认起始位置）到 endpos 结束位置（默认为 len(string)）进行匹配。

    • 当 整个字符串都匹配成功，返回 Match 类的实例对象（该实例对象包含匹配相关的信息：起始和结束位置、匹配的子串等等），否则返回 None。

  • 示例：

2.3.4 Pattern.findall 方法

  • 格式：Pattern.findall(string[, pos[, endpos]])，用于从字符串 string 的 pos 起始位置（默认起始位置）到 endpos 结束位置（默认为 len(string)）从左往右进行扫描，找到所有 不重复匹配，以列表的形式返回，如果有子组，那么 只保留子组的捕获内容，如果子组多个（至少两个子组），则以元组形式构建列表，如果没有找到匹配的，则返回空列表。

    • 示例：

2.3.5 Pattern.split 方法

  • 格式：Pattern.split(string, maxsplit=0) 用于将字符串 string 按照 Pattern 匹配到的子串进行分割，并以列表形式返回。

    • maxsplit：最大分割次数，默认为 0，表示不限制次数。

    • 如果有捕获分组，那么分组里匹配的内容也会包含在结果中。

2.3.6 Pattern.sub 方法

  • 格式：Pattern.sub(repl, string, count=0) 用于将字符串 string 中的匹配项替换为 repl。

    • count：模式匹配后替换的最大次数，默认 0 表示替换所有的匹配。
