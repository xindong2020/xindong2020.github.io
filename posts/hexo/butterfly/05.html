二、搜索

  • butterfly 主題支持三种搜索方式（algolia_search / local_search / docsearch），本站使用 Algolia 搜索。

2.1 Algolia 搜索

2.1.1 注册账号

  • Algolia官网（ https://www.algolia.com/ ）

  • 在项目根目录执行以下代码下载第三方包：

  • 在 Algolia 注册页面 使用 Google 账号、Github 账号或邮箱直接进行注册。

  • 登录后进入控制台，按以下步骤创建 index，自定义 indexName。

  • 在控制台进入设置页面，点击 API Keys 获取 Application ID、Search API Key、Admin API Key

  • 在 _config.yml 全局配置文件末尾加入以下内容：

    • content:strip:truncate,0,500

      • content：文章的内容。

      • strip：去掉 HTML 标签和样式。

      • truncate,0,500：截取内容的前 500 个字符。

    • excerpt:strip

      • excerpt：文章的摘要。

      • strip：去掉 HTML 标签和样式。

    • gallery：文章的图片集字段，不做处理直接索引。

    • permalink：文章的永久链接，不做处理直接索引。

    • slug：文章的短链接，不做处理直接索引。

    • title：文章的标题，不做处理直接索引。

  • 把主题配置文件 _config.butterfly.yml 中 search 的 use 设置为 algolia_search。

三、分享系统

  • butterfly 主题支持两种分享方式，通过设置 use，选择分享方式，可选 sharejs 或 addtoany，如果不提供分享功能则留空。

四、评论系统

  • Butterfly 主题支持多种评论系统：Disqus / Disqusjs / Livere / Gitalk / Valine / Waline / Utterances / Facebook Comments / Twikoo / Giscus / Remark42 / Artalk。可以根据自己的喜好选择其中一种评论系统。
  • 可以配置双评论系统，只需要同时配置两个评论系统（第一个为默认显示）。
  • 本站使用 Giscus 评论系统。

4.1 通用配置

  • use：使用的评论（请注意，最多支持两个，如果不需要请留空）。
    • 注：双评论不能是 Disqus 和 Disqusjs 一起，由于其共用同一个 ID，会出错。
  • text：是否显示评论服务商的名字。
  • lazyload：是否为评论开启 lazyload，开启后，只有滚动到评论位置时才会加载评论所需要的资源（开启 lazyload 后，评论数将不显示）。
  • count：是否在文章顶部显示评论数。Livere、Giscus 和 Utterances 不支持评论数显示。
  • card_post_count：是否在首页文章卡片显示评论数。Livere、Gitalk、Giscus 和 Utterances 不支持评论数显示。

4.2 Giscus

4.2.1 选择一个仓库

  • 一般选择博客本身即可，比如这里我用的就是这个 xindong2020/xindong2020.github.io 。

4.2.2 安装 giscus

  • https://github.com/apps/giscus 进入 giscus 的安装界面。

  • 点击 install，会提示选择一个仓库，这里就选择上一步中指定的仓库即可，后续 giscus 就会从该仓库读取数据。

4.2.3 开启 Discussions

  • 打开之前选择的仓库（xindong2020/xindong2020.github.io），进入 Settings 设置界面，勾选上 Discussions 以开启该仓库的 Discussions。

4.2.4 从 giscus 官网获取配置信息

  • 完成上述准备工作后就可以访问 giscus 官网 获取配置信息了。

  • 配置完成后，页面往下滑，会生成一个配置文件。

    • 主要关注以下字段：
      • data-repo
      • data-repo-id
      • data-category
      • data-category-id
      • data-mapping

4.2.5 在 Butterfly 主题中配置

  • 在主题配置文件 _config.butterfly.yml 中找到 giscus 配置项，根据上一步中重点关注的字段，对应填入内容。

4.2.6 汉化

  • 打开文件 themes\butterfly\layout\includes\third-party\comments\giscus.pug，添加内容 'data-lang': 'zh-CN',。

五、在线聊天

  • Butterfly 主题内置了多种在线聊天工具。你可以选择开启一种，方便你与访客的交流。

5.1 通用配置

  • use：选择你要使用的聊天工具，可选择 chatra / tidio / crisp。
  • rightside_button：是否开启右下角聊天按钮。
  • button_hide_show：是否开启滚动时隐藏聊天按钮。

5.2 chatra

  • https://chatra.com/cn/ 注册 chatra 并进行配置（需要使用国外邮箱地址），可以在 Preferences 中找到 Public key 填入 chatra 的 id 中。

5.3 tidio

  • https://www.tidio.com/ 注册 tidio 并进行配置，可以在 Preferences -> Developer -> Project data 中找到 Public key 填入 tidio 的 public_key 中。

5.4 crisp

  • https://crisp.chat/en/ 注册 crisp 并进行配置，找到需要的网站 ID 填入 crisp 的 website_id 中。
