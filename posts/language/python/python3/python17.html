一、可迭代对象

1.1 概述

  • 在 Python 中，任意实现了 迭代协议（__iter__ 方法）或 序列协议（__getitem__方法）的对象（Python 一切皆对象）都是可迭代对象（Iterable）。

  • 我们把可以通过 for...in... 语句迭代读取每一条数据元素的对象也称之为可迭代对象。

  • 可迭代对象不一定会是迭代器。

  • 常见的可迭代对象（不是迭代器）：字典（dict）、集合（set）、序列（list、tuple、str）、range 对象。

1.2 判断是否为 Iterable

  • 使用 isinstance(self, Iterable) / issubclass(cls, Iterable) 函数检测一个实例对象（self）/ 类对象（cls）是否为可迭代对象的方式 不是十分准确。

    • 不准确原因：isinstance 和 issubclass 判断一个对象是否为可迭代对象只检查 __iter__ 方法是否被实现，而不会检查 __getitem__ 方法。

1.4 __iter__ 方法

  • __iter__(self) 方法返回一个特殊的迭代器对象，这个迭代器对象实现了 __next__(self) 方法并通过 StopIteration 异常标识迭代的完成。

  • 在对一个对象进行迭代操作时（例：for...in...），会自动调用 __iter__ 方法，返回一个迭代器对象，然后每一次迭代，都会使用该迭代器对象调用一次 __next__ 方法，直到抛出 StopIteration 异常。

  • Python 语言内部会自动处理 for 循环和其他迭代上下文中的 StopIteration 异常。

    • 迭代上下文包括for循环，列表推导，内置in运算符，内置函数sorted,sum,any,all，除此之外还包括list,tuple,字符串join方法以及序列赋值运算，所有这些都使用了迭代协议来一次处理一个元素。

1.4 __getitem__ 方法

  • __getitem__ 方法可以让对象实现 迭代功能，返回与指定键相关联的值。

    • 对于序列来说，键应该是整数类型。

    • 对于映射来说，键可以是任何类型。

  • 在对一个对象进行迭代操作时（例：for...in...），会自动调用 __iter__ 方法。如果对象没有实现 __iter__ 迭代协议，Python 解释器会去寻找 __getitem__ 方法（序列协议）来迭代对象，如果 __getitem__ 没有被实现，解释器就会抛出异常 TypeError: 'xxx' object is not iterable。

  • 如果一个类实现了 __getitem__(self, item) 方法，每一次迭代都会自动调用 __getitem__ 方法，item 从 0 开始依次递增，直到发生 IndexError 为止。

二、迭代器

  • 在 Python 中，一个对象是迭代器，那它一定是可迭代对象，反之不然。一个对象拥有 __iter__ 方法就能够成为可迭代对象，但是迭代器必须实现 迭代器协议，即 同时实现 __iter__ 和 __next__ 方法。

    • 一个对象是迭代器，那它一定实现了 __iter__方法，实现了 __iter__方法就一定是可迭代对象。

  • 常见的迭代器：reversed、zip、enumerate、filter、map。
