Python3 错误和异常

一、语法错误

  • Python 的 语法错误 又称解析错误，也就是 解析代码时 出现的错误。当代码不符合 Python 语法规则时，Python 解释器在解析时就会报出 SyntaxError 语法错误，与此同时还会明确指出最早探测到错误的语句。例：

    • Python 解析器会打印出存在语法错误的代码行，并用小“箭头”指向行内检测到的第一个错误。

    • 本例中，在 print() 函数前检测到错误，是因为在它前面缺少了一个冒号（:）。

    • 错误信息 还会输出错误所在的文件名与行号，让我们知道去哪里排查错误。

  • PyCharm 是一款功能强大的 Python 集成开发环境（IDE），它内置了语法检查功能，能够帮助我们及时发现和修复代码中的语法错误。在 PyCharm 中，如果代码存在语法错误，它会立即在代码行下方显示一个红色的波浪线并显示错误信息。

  • 常见语法错误：

    • SyntaxError: invalid syntax（无效语法），造成这种错误的原因通常是遗漏了标点符号、关键字拼写错误或遗漏、变量名或函数名使用了关键字等。

    • SyntaxError: invalid character in identifier（标识符中有无效字符），造成这种错误的原因通常是代码中出现了无法识别的字符，检查一下是否有多余字符或中文字符。

    • SyntaxError: EOL while scanning string literal（检查到不完整的字符串），造成这种错误的原因通常是遗漏了字符串两边的引号，或两边的引号不统一。

二、异常

2.1 什么是异常？

  • 即使 Python 程序在语法上都是正确的，但是在 运行时 仍可能发生错误，这种在 运行时 产生的错误被称之为 异常（Exceptions）。

  • Python 集成开发环境（IDE）的语法检查功能不会发现异常。

  • 大多数的异常都不会被程序处理，如果程序在运行时出现了异常，默认情况下，程序终止并打印错误信息。例：

    • 错误信息的最后一行说明程序遇到了什么类型的错误（异常）。

    • 异常有不同的类型，这些类型都会作为错误信息的一部分打印出来，上述示例打印出的错误类型为 ZeroDivisionError。

    • 作为异常类型打印的字符串是内置异常的名称。

2.2 内置异常

  • 在 Python 中，所有异常类型都继承自 Exception 类，Exception 类又继承自 BaseException 类。Python 中有许多内置的异常类来表示某种具体异常。

  • 内置异常类型定义在 builtins 模块中，使用 dir(builtins) 可以查看。内置异常名称通常使用 Error 结尾。

  • 常见内置异常类型：

    • TypeError：类型异常，在对不同类型数据之间进行非法操作时引发此异常。

    • ZeroDivisionError：除零异常，除法运算中除数为 0 时引发此异常。

    • KeyError：键异常，访问字典中不存在的键时引发此异常。

    • ValueError：值异常，当传入无效参数时引发此异常。

    • IndexError：索引异常，索引超出序列范围时引发此异常。

    • AttributeError：属性异常，访问的对象属性不存在时引发此异常。

    • NameError：名称异常，访问一个未声明的变量时引发此异常。

    • RecursionError：递归异常，当递归深度超过 1000 时引发此异常。

2.3 异常处理

  • 如果不想要程序在出现异常时终止运行，可以使用捕获异常的方式捕获这个异常，然后对异常进行处理，处理后让程序继续运行，这种捕获异常并处理的方式叫作异常处理。

  • 异常处理可以使用 try except 相关语句。

2.3.1 try except 语句

（1）语法结构

  • Python 中，捕获并处理异常可以使用 try except 语句，语法结构：

    • 使用 try 子句将可能产生异常的代码块包裹，以便后续捕获异常。

    • 一个 try except 语句可以包含 多个 except 子句，用来处理不同类型的异常。

    • 多个 except 子句 最多只有一个 会被 执行。

    • Error1、Error2、Error3 和 Error4 表示异常类型的名称，一个 except 子句可以处理 多个 异常，多个异常类型必须放在 元组 中。

    • [as e] 可选，用于将捕获到的异常对象 赋值 给 e（e 可以自定义，符合标识符规定即可），方便在 except 中使用。

      • e 相当于异常类型的实例对象，我们可以通过 e 使用这个异常类型的的所有属性和方法。

    • [Exception] 可选，表示可以忽略异常的名称，通常用在最后一个 except 块。

  • 示例：

    def func(x, y):
    try:
        print(x / y)
    except TypeError:
        print("类型异常")
    except ZeroDivisionError:
        print("除零异常")
    except:
        print("end")
        
func("1", "a")  # 类型异常
func(2, 0)  # 除零异常
func(9, 3)  # 3.0
（2）执行流程

  • try except 语句的执行流程：

    • 首先，执行 try 子句。

      • 如果没有异常发生，忽略 except 子句，try 子句执行后结束。

      • 如果发生异常，系统会自动生成一个异常类型，引发异常的代码行之后的语句会被跳过。

    • 发生了异常，将系统生成的异常类型与 except 子句中指定的异常类型进行比对。

      • 如果系统生成的异常类型和某个 except 字句中指定的异常类型 相匹配，那么异常就被成功 捕获，对应的 except 子句开始 处理异常。

      • 如果这个异常类型没有与任何的 except 子句匹配，则程序终止运行。

（3）嵌套

  • 在 Python 中，try except 异常处理语句也可以嵌套。当内层 try except 出现异常，且指定的异常类型与系统生成的异常类型 不匹配 时，向外层 try except 传递。如果与外层指定的异常类型 相匹配，则异常 被处理，否则继续向外传递，直至最外层，如果系统生成的异常类型与所有指定的异常类型 都不匹配，则 程序终止。

    • 注：except 子句（嵌套的 try except）中触发的异常，不能 被同级其他的 except 子句捕获，因为多个 except 子句 最多只有一个 会被 执行。

  • 示例：

    try:
    print("第 1 层 try")
    try:
        print("第 2 层 try")
        print("dong"[7])
    except IndexError:
        print("第 2 层 except")
        try:
            print("第 3 层 try")
            print("2" + 1)
        except TypeError:
            print("第 3 层 except")
            print(2 / 0)
    except ZeroDivisionError:
        print("第 2 层 except 异常: ZeroDivisionError")
except ZeroDivisionError:
    print("第 1 层 except 异常: ZeroDivisionError")

# 输出：
#   第 1 层 try
#   第 2 层 try
#   第 2 层 except
#   第 3 层 try
#   第 3 层 except
#   第 1 层 except 异常: ZeroDivisionError

- **疑惑**：为什么第 `3` 层 `except TypeError` 中 `print(2 / 0)` 触发的 `ZeroDivisionError` 异常，**没有** 被 **第** `2` **层** 的 `except ZeroDivisionError` 捕获，反而被 **第** `1` **层** 的 `except ZeroDivisionError` **捕获** 到了？

- **解惑**：`print(2 / 0)` 触发异常 `ZeroDivisionError`，它所在的位置是第 `2` 层 `except IndexError` 中的嵌套 `try except` 语句中，根据 **多个** `except` **子句最多只有一个会被执行的原则**，同层级（第 `2` 层）的 `except ZeroDivisionError` 不会被执行，所以异常向外传递到第 `1` 层，被第 `1` 层的 `except ZeroDivisionError` 捕获。
（4）获取异常的相关信息

  • 在 Python 中，每种异常类型都提供了下列属性和方法，方便我们获取当前异常类型的相关信息。

    • e.args：返回一个元组类型，其中的元素是该异常的参数。

    • str(e)：返回异常信息，但不包括异常信息的类型。

    • repr(e)：返回较全的异常信息，包括异常信息的类型。

  • 示例：

    def func(x, y):
    try:
        print(x / y)
    except (TypeError, NameError, ZeroDivisionError) as e:
        print(e)  # division by zero
        print(e.args)  # ('division by zero',)
        print(type(e))  # <class 'ZeroDivisionError'>
        print(str(e))  # division by zero
        print(repr(e))  # ZeroDivisionError('division by zero')

func(2, 0)

2.3.2 try except else 语句

  • try except 语句之后可以紧跟一个 else 子句，这个 else 子句必须放在所有的 except 子句之后，并且 else 子句仅会在 try 子句 没有 发生任何 异常 时被 执行。

    • 注：else 子句中触发的异常，不能被同级其他的 except 子句捕获。

  • 语法结构：

    try:
    可能产生异常的代码块
except [(Error1, Error2, ...) [as e]]:
    处理异常的代码块 1
except [(Error3, Error4, ...) [as e]]:
    处理异常的代码块 2
except [Exception]:
    处理其它异常
else:
    没有异常时执行的代码块

  • 示例：

    def func(x, y):
    try:
        print(x / y)
    except TypeError:
        print("类型异常")
    except ZeroDivisionError:
        print("除零异常")
    except:
        print("end")
    else:
        print("无异常")

  func("1", "a")  # 类型异常
  func(2, 0)  # 除零异常

  # 输出：

#   

# 3.0

#   无异常

  func(9, 3)  # 此次调用不触发异常，else 子句可以被执行

2.3.3 try finally 语句

  • try 子句后可以紧跟一个 finally 子句（可以没有 except 子句）。无论 try 子句中是否发生异常，最终都会进入 finally 子句，并执行其中的代码块。

  • 基于 finally 子句的这种特性，在某些情况下，当 try 子句中的程序打开了一些物理资源（例：文件、数据库连接等）时，由于这些资源必须手动关闭，通常这些关闭资源的工作就放在 finally 子句中。

  • 当 try 子句和 finally 子句中都包含 return 语句时，只有 finally 子句中的 return 被返回。

  • finally 子句只要求和 try 子句搭配使用，而 try 子句后是否包含 except 子句和 else 子句，对于 finally 不是必须的（else 子句必须搭配 try except 语句使用）。

    • 如果 try 子句后包含 except 子句和 else 子句，finally 子句必须放到整个语法结构的 末尾。

  • 示例：

    def func(x, y):
    try:
        x / y
    except ZeroDivisionError:
        print("除零异常")
    except:
        print("end")
    else:
        print("else 子句只有不发生异常，才会执行")
    finally:
        print("finally 子句无论是否异常，都会执行")
print("2 / 0")
func(2, 0)
print("2 / 2")
func(2, 2)

# 输出：

  #   2 / 0

  #   除零异常

  #   finally 子句无论是否异常，都会执行

  #   2 / 2

  #   else 子句只有不发生异常，才会执行

  #   finally 子句无论是否异常，都会执行

2.4 抛出异常

  • Python 允许我们使用 raise 语句抛出一个指定的异常，以及异常信息的相关描述。

  • raise 语句的语法格式：raise [Exception]。

    • Exception：异常类型，它必须是一个异常类的 实例对象 或异常 类本身（Exception 的子类）。

    • args：异常的描述信息。

  • raise 语句的用法：

    • raise：单独一个 raise 关键字，可选参数全部省略，raise 会把当前错误原样抛出。默认抛出 RuntimeError 异常。例：

      def func(x, y):
    try:
        try:
            x / y
        except ZeroDivisionError:
            raise  # 将 ZeroDivisionError 原样抛出
    except ZeroDivisionError as z:
        print(z)
func(2, 0)  # division by zero

    • raise Exception：raise 后带一个异常类的 实例对象 或异常 类本身，表示抛出该类型的异常。

      • raise 异常类本身时，没有异常的描述信息，仅有触发的异常类型。

      • raise 异常类的实例对象时，可以初始化异常类的基本属性，例：异常信息。

      • 示例：

        def func(x, y):
    try:
        if y == 0:
            raise ZeroDivisionError(f"{x} / {y}，除数不能为 0")  # 抛出异常类的实例对象
        elif not x.isdigit():
            raise ValueError  # 抛出异常类本身
    except ZeroDivisionError as z:
        print(repr(z))
    except ValueError as v:
        print(repr(v))

func(2, 0)  # ZeroDivisionError('2 / 0，除数不能为 0')
func("a", 10)  # ValueError()

2.5 自定义异常

  • Python 中内置了大量的异常类，虽然这些异常类可以描述编程时出现的绝大部分情况，但仍难以涵盖所有可能出现的异常。Python 允许程序开发人员自定义异常。

  • 自定义异常的方法很简单，只需创建一个类，让它 继承 Exception 类或其它异常类即可。

  • 自定义异常类与普通类一样，也可以具有自己的它属性和方法，但一般情况下不添加或者只为其添加几个用于描述异常的详细信息的属性即可。
