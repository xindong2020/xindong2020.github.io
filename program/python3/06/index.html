Python3 深拷贝和浅拷贝

一、赋值

1.1 id() 函数

  • 格式：id([object])，内置函数，返回 object 对象的唯一标识符（内存地址），是一个整数。注：每次程序运行返回的内存地址都会不同。

    • 变量名存储的只是一种引用关系，所以通过 id 函数获取变量地址时，实际是返回了值的地址。

1.2 小整数对象池

  • 整数在程序中的使用非常广泛，Python 为了优化速度，避免为整数频繁申请和销毁内存空间，使有了小整数对象池。

  • Python 对小整数的定义是 [-5, 256]，这个范围内的整数对象都是提前建立好的，不会被垃圾回收。在一个 Python 的程序中，所有位于这个范围内的整数使用的都是 同一个对象。

    • 为了节省内存空间，解释器（例：PyCharm）做了优化，如果直接在解释器中执行程序文件，相同值的变量会引用自同一地址，而在交互式窗口运行则可以看到区别。

1.3 大整数对象池

  • 与小整数对象不同，如果是相同值的大整数对象，在内存里面不是同一个地址，也就是说是不同的两个对象，不会让它们共享引用，而是重新开辟一个内存空间存放。但是，如果处于 相同代码块 的大整数对象，它们是 同一个内存地址（同一对象），会让其共享引用，不会再开辟一个内存空间存放。例：

1.4 字符串的 intern 机制

  • intern 机制： 如果两个及以上的字符串变量，它们的值相同且仅由数字、字母 和 下划线 组成，并且长度不超过 20 个字符；或者字符串的值仅含有 一个字符 时，内存空间中只创建一个对象来让这些变量都指向该内存地址（共享引用）。当字符串不满足该条件时，相同值的字符串变量在每次创建时都会申请一个新的内存地址来保存值。

1.5 Python 赋值

  • Python 中, 变量的定义和赋值必须是同时进行的，赋值只是将一个对象的 内存地址 赋值给了一个变量，让变量指向该地址，而不是将数据值赋给了变量。

  • 当执行程序 num = 77 时，Python 解释器做的事情：

    • 创建变量 num。

    • 创建一个对象（分配一块内存），来存储值 77。

    • 将变量与对象连接起来，从变量到对象的连接称之为引用（变量引用对象）。

  • Python 中的赋值语句不复制对象，只是建立 引用关联。

  • 严格来说，变量名本身没有类型，通常我们所说的变量类型指的是值的数据类型。

  • 为了节省内存空间，解释器（例：PyCharm）做了优化，如果直接在解释器中执行程序文件，相同值（不可变类型）的变量会引用自同一个对象，而在交互式窗口运行则可以看到区别。

  • 当一个可变类型的数据被多个变量名引用时，如果对任意一个变量名进行修
    改，那么该变量名所指向的原数据就会被修改，所有引用这个原数据的变量都会发生改变。例：

二、浅拷贝

  • 浅拷贝指的是重新分配一块内存，创建一个新的对象，但里面的元素是原对象中各个子对象的 引用。

    • 赋值不是浅拷贝，赋值来的对象就是完完全全的原始对象，只是叫的名字不同而已。

  • 浅拷贝有三种形式：

    • 使用数据类型本身的 构造器，例：list()、tuple()、str() 等。

    • 对于序列，可以通过 完全切片操作 [:] 来完成浅拷贝。

    • Python 还提供了 copy 模块 中的 copy.copy(obj) 函数，适用于任何数据类型。

      • list.copy()、dict.copy() 和 set.copy() 都属于浅拷贝。

  • 浅拷贝之所以称为浅拷贝，是因为它仅仅只拷贝了一层，浅拷贝出来的对象就是 外新内旧 的对象，对象本身（id）和原始对象完全不同，但是子对象和原始对象的子对象是一样的。

    • 如果最外层的数据类型是 不可变 的（数值，字符串，元组），则 不发生拷贝，这种情况相当于赋值，新对象与被拷贝对象的 id 值相同。例：

    • 如果最外层的数据类型是 可变 的（列表、字典、集合），则（最外层）发生拷贝，创建一个 新对象，但是内部元素不发生拷贝，新的对象中保存的只是对原对象中数据的引用。例：

三、深拷贝

  • 深拷贝指的是重新分配一块内存，创建一个新的对象，并且将原对象中的所有元素，包括多层嵌套的元素，全部拷贝到新对象中。

  • 深拷贝出来的对象是一个全新的对象，修改原对象或新对象时，它们之间互不影响。

  • 由于深拷贝需要维护一块内存用于记录已经拷贝的对象，所以深拷贝的运行速度会比浅拷贝 慢。

  • 在 Python 中，深拷贝 只有一种形式：copy 模块 中的 copy.deepcopy(obj) 函数，适用于任何数据类型。

  • 深拷贝的原则：不管子对象多深，只要可变，都会拷贝成一份新对象。

  • 深拷贝的规则：

    • 如果被拷贝的对象 本身不可变（数值，字符串，元组）且 包含的所有子对象都不可变，则该对象 不发生拷贝。例：

    • 如果被拷贝的对象 本身可变（列表、字典、集合）或 本身不可变但包含有可变的子对象（例：元组包含列表），则该对象 发生拷贝。例：

    • 对于被拷贝的对象包含的 子对象，在拷贝时依然 遵循上两条规则。例：

      • 子对象 [1, 2] 列表本身可变，发生拷贝，内部的数字 1 和 2 不可变，不发生拷贝。

      • 数字 3、字符串 "hello"、元组 (5, (6, 7)) 本身不可变，不发生拷贝。

      • 子对象 ([7, 8], 9) 元组本身不可变，但包含可变的子对象 [7, 8] 列表，发生拷贝。

        • 元组中的子对象 [7, 8] 列表本身可变，发生拷贝，内部的数字 7 和 8 不可变，不发生拷贝。

      • 子对象 {"name": "dong", "language": ["Java", "Python"]} 字典本身可变，发生拷贝。

        • 字典的键都是不可变类型，因此不发生拷贝。

        • 键 "name" 对应的值 "dong" 为字符串不可变，不发生拷贝。

        • 键 "language" 对应的值 ["Java", "Python"] 为列表可变，发生拷贝。列表中的元素都为字符串，不发生拷贝。

      • 子对象 {"a", "b"} 集合本身可变，发生拷贝，内部的字符串 "a" 和 "b" 不可变，不发生拷贝。
